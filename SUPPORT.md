## üöÄ Common Questions

### General Questions

**Q: What is The Pink Revolution?**
A: A transformation initiative to change all blue colors to pink across overkor-tek repositories, which evolved into a comprehensive organizational improvement project.

**Q: How can I contribute?**
A: See the [Contributing Guidelines](./CONTRIBUTING.md) and [Quick Start Guide](./docs/QUICK_START.md). Start with documentation improvements or small fixes!

**Q: Where do I start?**
A: Read the [Quick Start Guide](./docs/QUICK_START.md) first, then explore the [Documentation Index](./docs/INDEX.md).

**Q: What repositories are part of this?**
A: Check the [Master Plan](./docs/OVERKOR-TEK_MASTER_PLAN.md) for the complete list of overkor-tek repositories.

### Technical Questions

**Q: Why pink instead of blue?**
A: "One thing became a whole bunch" - D. It started as a simple preference and became a symbol of organizational transformation.

**Q: How do I fix a typo?**
A: Click the pencil icon on any file, make your change, and submit a pull request. Small fixes don't need an issue first.

**Q: Can I reorganize documentation?**
A: Yes, but open an issue first to discuss major structural changes with maintainers.

**Q: How often should I update the Action Checklist?**
A: Update it whenever you complete a task. Don't forget to also update the [CHANGELOG.md](./CHANGELOG.md).

### Process Questions

**Q: How long does review take?**
A: Small PRs (typos, links) usually reviewed within 24-48 hours. Larger changes may take longer.

**Q: Who approves changes?**
A: Repository maintainers review and approve changes. See the [Master Plan](./docs/OVERKOR-TEK_MASTER_PLAN.md) for team structure.

**Q: What if my PR is rejected?**
A: Maintainers will provide feedback. You can make changes and resubmit. Rejection is rare and always includes explanation.

**Q: How do I become a maintainer?**
A: Consistent, quality contributions over time. Maintainers are added based on trust and expertise.

## üêõ Found a Security Issue?

**DO NOT** open a public issue for security vulnerabilities.

Instead:
1. Review the [Security Policy](./SECURITY.md)
2. Report privately through GitHub Security tab
3. Or contact repository maintainers directly

We take security seriously and will respond promptly.

## üìä Response Times

We aim for these response times (though we're a small team):

- **Critical bugs:** 24 hours
- **Security issues:** 48 hours
- **General questions:** 2-3 business days
- **Feature requests:** 1 week
- **Documentation PRs:** 2-3 business days
- **Code PRs:** 1 week

*Note: These are goals, not guarantees. We're all human! üíñ*

## üéØ Help Us Help You

When asking for help, please provide:

1. **Context** - What are you trying to do?
2. **What you've tried** - What have you already looked at?
3. **Specific questions** - Be clear about what you need
4. **Screenshots** - If relevant, show what you're seeing
5. **Links** - Reference specific files or sections

### Good Question Example
```
I'm trying to understand the repository structure for overkor-tek.
I've read the README and Master Plan, but I'm unclear about
the difference between Tier 1 and Tier 2 repositories.

Specifically, what criteria determine the tier assignment?

I checked the Master Plan (docs/OVERKOR-TEK_MASTER_PLAN.md)
but couldn't find this information.
```

### Less Helpful Question
```
How does this work?
```
